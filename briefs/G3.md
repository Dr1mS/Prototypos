# G3 — The memory ladder: which architectural property produces behavioral attractors?

**Addressed to the supervisor session (Fable 5).** Reuses ALL G2/G2.5 infrastructure
(agent loop, probe battery, judge, the 12 frozen Exp-2 orderings, the random-walk

## 0. Why this exists (the reviewer's one objection, pre-empted)

G2.5's spine result — natural memory shows no attractor structure; engineered
explicit latent state does — rests on **two points** (one natural design vs one
engineered design). The obvious rebuttal: *"you tested one memory architecture; a
more sophisticated memory would drift."* G3 answers it by running a **ladder of
memory architectures** through the identical protocol and asking which
**architectural property** (not which memory sophistication) produces attractors.

The honest ceiling, stated up front: a ladder gives **evidence for necessity**, never
proof — we cannot test every architecture. The claim it licenses is *"across a
graded family of memory architectures, only [property X] produces attractors; memory
sophistication without it does not"* — much harder to attack than the two-point
version, and it locates the *mechanism*.

## 1. PRE-REGISTRATION (`prereg_g3.md`, supervisor writes + git-commits BEFORE any run)

Freeze predictions, the axis definition, and the interpretation rule before data.
No post-hoc relabel (the G1/G2.5 rule).

### 1.a Define the "explicit state" axis FIRST (the anti-gradient-trap)

The boundary "memory vs explicit state" is NOT clean — a memory that compresses to a
compact recurrent summary *starts to be* a latent state. So do not pre-assume a
binary. Classify every rung on three independent properties, in a frozen table,
before running:

- **Compression**: is the state low-dimensional and bounded (a scalar / few numbers)
  or unbounded growing text?
- **Recurrence**: is the state re-read and re-written each step, feeding back on
  itself, or append-only?
- **Perception-coupling**: does the state condition the *interpretation of new input*
  (the G1 mechanism — biasing appraisal) or only retrieval / expression?

G0–G1.5 identified perception-coupling as the active ingredient. **The pre-registered
hypothesis: attractor formation tracks perception-coupling specifically, not
compression or recurrence alone, and not memory sophistication.**

### 1.b The rungs, classified (freeze this table)

| rung | compression | recurrence | perception-coupling | status |
|---|---|---|---|---|
| R0 memoryless | — | none | none | **have** (G2 control) |
| R1 raw log | none (grows) | append-only | none | **NEW** |
| R2 summarized | medium | yes (rewrite) | weak/implicit | **have** (G2 natural agent) |
| R3 vector | none (grows) | append + top-k retrieve | none | **NEW** |
| R4 explicit latent | high (low-D) | yes | **strong** | **have** (G0/G1.5) |
| R2.5 coupled-summary (optional bridge) | medium | yes | **injected** | optional NEW |

R3 (vector) is the **sharp test**: it is the "more sophisticated memory" the reviewer
demands, but it has NO perception-coupling. If R3 shows no more path-dependence than
R2/R1, "sophistication" is ruled out and perception-coupling is implicated. The
optional R2.5 (a summarized memory whose summary is fed back as *interpretation
guidance*, adding perception-coupling to R2) locates the threshold — add it only if
you want to pin the mechanism precisely.

### 1.c Predictions + interpretation rule

- Per-rung prediction: which rungs show path-dependence beyond the random-walk null.
  Seed: only R4 (and R2.5 if run). R0/R1/R2/R3 do not.
- **Interpretation rule (commit both branches):**
  - **Clean wall** (only perception-coupled rungs pass) → "attractor formation
    requires perception-coupling; memory persistence/sophistication is insufficient."
    The strong version of the spine.
  - **Gradient** (path-dependence rises smoothly with one property) → report the
    gradient and name the axis it tracks; this is a *better* result — it forces a
    definition of "explicit state" and is a mechanistic contribution in itself. Be
    ready for this; do not force a wall.

  use caution — both are "final position on the behavioral axis, spread across
  orderings", so `std(final)` is the comparable path-dependence metric. State this in
  the report; do not compare absolute levels across substrates.

## 3. SUBAGENT A (Opus 4.8) — the two new memory backends

Deliver R1 (raw log) and R3 (vector), each conforming to the **existing agent-loop
interface** (drop-in for the memory component; the agent/probe/judge are untouched —
anti-duplication).
- **R1 raw log**: append every turn verbatim; retrieve recent-N (or all) into context;
  no summarization, no rewrite.
- **R3 vector**: embed each turn (a local embedding model), store, retrieve top-k by
  similarity into context; no summarization, no self-rewrite, no scalar state.
- Per-backend **coherence gate** (G2.5 §2.d): does the agent follow the loop and
  produce parseable probes with this backend? Log pass rate. R3 especially: confirm
  retrieval actually returns relevant memories (a broken retriever = a confound).
